## Overview

Scout AI is a soccer player performance analysis tool.
Users can input match data, view historical stats, and get ML-generated performance predictions.

---

## Business rules

- A player belongs to a team and has a position (goalkeeper, defender, midfielder, forward)
- A match always has two teams and a date
- Each player can have multiple events per match (goals, assists, passes, etc.)
- Performance rating goes from 0.0 to 10.0
- Predictions are generated by an ML model and stored alongside the actual rating for future comparison
- Historical match data is the main training source for the model


## API — expected endpoints

```
GET  /players                    → list all players
GET  /players/:id                → player detail + last 5 events
POST /players                    → create new player

GET  /matches                    → list matches (paginated)
POST /matches                    → insert match + player events in one transaction

POST /predict                    → receives player_id, returns predicted rating
GET  /predictions/:player_id     → prediction history vs actual ratings
```

## Code conventions

### Python (api/)

- Explicit typing everywhere (`from __future__ import annotations`)
- Pydantic for all input validation and response serialization
- SQLAlchemy with sessions via dependency injection in FastAPI (`Depends(get_db)`)
- Never return ORM objects directly — always serialize via Pydantic schema
- HTTP errors via `HTTPException` with English messages

### TypeScript (web/)

- Functional components with explicit prop interfaces
- All API calls centralized in `lib/api.ts` — no component calls `fetch` directly
- Component names in PascalCase, file names in kebab-case
- No `any` — if the type is unknown, create an interface

### Database (db/)

- Table names in English, plural, snake_case: `players`, `matches`, `player_events`, `predictions`
- Every table has `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
- Every table has `created_at TIMESTAMPTZ DEFAULT now()`
- Foreign keys always with `ON DELETE CASCADE`
